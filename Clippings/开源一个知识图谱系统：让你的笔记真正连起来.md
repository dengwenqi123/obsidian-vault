之前一直在用 [Obsidian](https://zhida.zhihu.com/search?content_id=273349770&content_type=Article&match_order=1&q=Obsidian&zhida_source=entity) 记笔记，笔记多了就发现一个问题： **它们都是孤岛** 。  
这篇笔记和那篇笔记有没有关系？靠记忆，早就忘了。  
所以，结合 [Karpathy LLM Wiki](https://zhida.zhihu.com/search?content_id=273349770&content_type=Article&match_order=1&q=Karpathy+LLM+Wiki&zhida_source=entity) 的思想，做了一个知识图谱系统—— **Obsidian- [Graphify](https://zhida.zhihu.com/search?content_id=273349770&content_type=Article&match_order=1&q=Graphify&zhida_source=entity) 技能。**  
实现昨天的承诺，今天开源了！

### 它能帮你做什么？

下面简单说说它的功能：  
**一张图看清所有知识**  
打开图谱，所有笔记变成节点，Wiki 链接变成连线。  
谁和谁有关系，一目了然。

![](https://pica.zhimg.com/v2-cc8a2c3dd2bfb4348a0551fb4b91da40_1440w.jpg)

**点击节点，直接打开笔记**  
Graphify 的所有节点可以点击，双击节点，直接打开那篇笔记，无缝衔接。  
**路径追踪：发现知识脉络**  
选两个节点，自动算出它们之间关联的最短路径。  
比如「AI-Agent」和「知识图谱」之间，可能经过「LLM Wiki」→「Obsidian」→「Graphify」。

![](https://picx.zhimg.com/v2-b9b3fb40d0c631d1dc2945f5adade34d_1440w.jpg)

**这条路，就是你知识的脉络。**  
孤岛检测：告诉你哪里要补链接  
没有连接的节点会特别显示，一眼就知道哪些笔记是孤岛，需要建立链接。

![](https://pica.zhimg.com/v2-93c3c5800e02b8d159ae929bdb942ca6_1440w.jpg)

**热度统计：你最关注什么**  
你点击最多的节点，面积最大、颜色最深，。  
一眼就能看出你最关注的领域。

![](https://pic3.zhimg.com/v2-f2d959aa517217dc4e800e591b046fbe_1440w.jpg)

**影响力分析：你的核心知识是什么**  
PageRank 式排名，知道哪个知识点连接最多。节点连接数量越多，节点显示就越大，颜色越深。

![](https://pic1.zhimg.com/v2-c0992144f516d9f0fb5b49f3b4edced2_1440w.jpg)

### 安装：让你的Agent 去干

上面这些功能，搭建起来要跑一堆命令、改配置、处理报错。  
现在我把完整流程封装成了一个 Agent Skill。你把仓库地址扔给你的Agent  
只需要对它说一句话：  
"安装这个技能，帮我搭建知识图谱系统"  
Agent 就会：

1. 问你 Obsidian vault 路径在哪
2. 检查环境有没有 Python
3. 自动安装依赖
4. 创建 `.graphify` 目录
5. 生成图谱 HTML 文件
6. 问你要不要启动自动更新

整个流程分 5 个阶段，每个阶段结束 Agent 都会停下来问你确认，不会乱搞。  
**不管是 [Hermes](https://zhida.zhihu.com/search?content_id=273349770&content_type=Article&match_order=1&q=Hermes&zhida_source=entity) 、 [Claude](https://zhida.zhihu.com/search?content_id=273349770&content_type=Article&match_order=1&q=Claude&zhida_source=entity) 、 [Cursor](https://zhida.zhihu.com/search?content_id=273349770&content_type=Article&match_order=1&q=Cursor&zhida_source=entity) 还是其他 Agent，只要能读取 Skill 文档，就能按流程干活。**
